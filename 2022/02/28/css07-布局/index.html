07-布局

发表于 2022-02-28 更新于 2022-04-02 分类于 CS ， CSS

一 文档流

  1. 标准流

    1. 特点
      1. 元素向右、向下排列
      2. 块级元素独占一行
      3. 行内级元素从左到右，空间不够自动折行
    2. 元素的层叠
      1. 父子元素：子元素层叠在父元素上
      2. 非父子元素
        1. 都是非定位元素：标准流中不存在层叠
        2. 1个定位1个非定位：定位元素层叠在非定位元素上
        3. 都是定位元素：z-index 控制层叠

  2. 脱离标准流

    1. 方式
      1. position: absolute/fixed
      2. float
    2. 特点
      1. 宽高默认由内容决定，且可以设置宽高
      2. 不再受标准流约束，不再向父元素报告宽高

  3. display、postion 和 float 的联系

    1. display：none，则盒子消失，position、float 无效

    2. 否则，当元素脱标，则 display 按照下表改变

      Specified value Computed value
      inline-table table
      inline, table-row-group, table-column, table-column-group, table-header-group, table-footer-group, table-row, table-cell, table-caption, inline-block block
      others same as specified

    3. 可以看到脱标元素会块级化

      1. 注：块级元素不是独占一行，是独占父元素一行
      2. 脱标元素是块级元素，只是表现的好像行内块元素一样，一行可以放多个

二 浮动

2.1 语法

  1. 早期浮动是为了实现图文环绕效果
  2. 现在多用于网页布局
    1. 让块级盒子排成一行，即垂直布局的盒子变成水平布局
    2. 盒子左右对齐

2.2 浮动的特性

  1. 浮动会使元素脱离标准流，漂浮在标准流上方且不再占据标准流位置，直到边缘触及到包含块或其它浮动元素的边缘
    1. 浮动元素不能超出包含块
    2. 浮动元素之间不会层叠
    3. 浮动元素的顶端不能超过父元素的顶端，也不能超过之前所有浮动元素的顶端
  2. 浮动元素不会与行内级元素层叠
  3. 浮动元素顶端会与所在行的顶端对齐
  4. 当一个元素浮动时
    1. 其上一行
      1. 是行内级元素，且空间
        • 能够放下浮动元素，则浮动元素会在上一行浮动直到边缘触及到包含块或另一个浮动元素
        • 放不下浮动元素，则浮动元素在当前行浮动
      2. 是块级元素
        • 块级元素独占父元素一行，即上一行没有浮动元素的位置
        • 浮动元素在当前行浮动
    2. 其下一行
      1. 是行内级元素
        • 会在合适的位置环绕浮动元素
      2. 是块级元素
        • 会无视浮动元素的存在而上移

2.3 清除浮动

1 语法

  1. clear 只能影响清除浮动的元素本身，无法影响其它元素
  2. clear 属性只对块级元素有效
  3. 默认 none
    1. left：要求设置 clear 的元素的顶部低于之前生成的所有左浮动元素的底部
    2. right：要求设置 clear 的元素的顶部低于之前生成的所有右浮动元素的底部
    3. both：要求设置 clear 的元素的顶部低于之前生成的所有浮动元素的底部

2 清除浮动的原因

  1. 由于浮动元素会影响后边正常文档流中的元素，所以通常会把浮动元素放在一个处于正常文档流的父元素中
  2. 父元素通常不会固定高度，而是根据子元素高度来撑开高度，而浮动元素脱离文档流，不占据文档流的位置，就会导致父元素高度塌陷
  3. 清除浮动就是为了解决父级元素因为子级浮动引起内部高度为 0 的问题

3 清除浮动的方式

方式 说明
1 父元素显式定义高度 height 不推荐，只适合高度固定时使用
2 父元素末尾添加一个空标签并清除浮动 不推荐，增加不必要的标签
3 父元素创建伪元素并清除浮动 推荐
4 父元素 overflow: hidden 父元素触发 BFC

2 BFC（Block Formatting Context）

1 BFC 概念
概念 说明
FC（formattion context） 格式化上下文，就是正常文档流
BFC（Block Formatting Context） 块级格式化上下文
IFC（inline formattion context） 行内格式化上下文
GFC（grid formattion context） 栅格格式化上下文
  • 块级格式化上下文，是 css 布局的一个概念，是一块区域，一个环境
2 BFC 触发条件
元素 说明
根元素 <html>
浮动元素 float: left / right
绝对定位元素 position: absolute / fixed
overflow 不为 visible overflow: hidden / auto / scroll
display display: inline-block / table-cell / table-caption / flex / inline-flex
3 BFC 布局规则
  1. 计算 BFC 的高度时，浮动元素也参与计算（用来清除浮动影响）
  2. BFC 会形成一个隔离的独立容器，容器中的子元素不会影响到外面的元素；外面的元素也不会影响到容器中的子元素
  3. BFC 中每个子元素的 margin-box 的左边会接触 BFC容器的左边，无论子元素是否浮动
  4. BFC 的区域不会与 float box 重叠
4 BFC 应用

  1. 清除浮动：

    1. BFC 盒子会默认包裹住内部子元素（不管是标准流子元素还是浮动子元素）
    2. 由于 BFC 容器计算高度会计算浮动元素，所以只需父元素触发 BFC 即可清除浮动

  2. 两栏自适应布局

    1. 左边元素浮动，右边元素触发 BFC即可，因为 BFC 的区域不会与 float box 重叠，如图文两栏布局
    2. 图文排列时，图片浮动，就形成文字环绕
    3. 如果想让图片在左，文字在右的两列布局，只需文字容器触发 BFC 即可

  3. 解决相邻元素 margin 合并

    1. 由于 BFC 与外界元素不会相互影响，所以只需相邻元素不在同一个 BFC 容器中即可
    2. BFC 盒子本身与子元素之间不存在 margin 塌陷

三 定位

3.1 语法

值 名称 基点 特点 元素块级化（行内块）
static 静态定位 正常标准流 没有边偏移，就是正常标准流中的元素 ×
relative 相对定位 相对于元素原来在标准流中的位置定位，不影响其它元素 占据原标准流位置（飞起来了但还占着原来位置） ×
absolute 绝对定位 相对于离元素最近的非静态定位的长辈元素（直到浏览器可视窗口）定位 不占据标准流位置，会随着滚动条滚动 √
fixed 固定定位（绝对定位的一种特殊形式） 相对于浏览器可视窗口定位 不占据标准流位置，不会随着滚动条滚动 √
sticky 粘性定位 占据标准流位置

1 粘性定位

  1. sticky会产生动态效果，很像relative和fixed的结合：一些时候是relative定位（定位基点是自身默认位置），另一些时候自动变成fixed定位（定位基点是视口）
  2. sticky生效的前提是，必须搭配top、bottom、left、right这四个属性中的一个或多个一起使用（看滚动方向），不能省略，否则等同于relative定位，不产生”动态固定”的效果
  3. 不能有任何祖先元素设置overflow:hidden，否则没有粘滞效果。因为改变了滚动容器（即使没有出现滚动条）
  4. 父元素的高度不能低于sticky元素的高度
  5. sticky元素仅在其父元素内生效，不会超出父元素，滚动时会随着父元素一起消失
  6. 当页面滚动，父元素开始脱离视口时（即部分不可见），只要与sticky元素的距离达到生效门槛（设置的top/bottom/left/right），relative定位自动切换为fixed定位，等到父元素完全脱离视口时，fixed定位恢复成relative定位
  7. 同一个父容器中的sticky元素，如果定位值相等，则会重叠；如果属于不同父元素，则会随着父元素不再完整占据sticky元素的固定区域以后，再由其他父元素的sticky子元素占据固定位置
  8. 我的理解就是设置了粘性定位的元素，没到达所设置的偏移量时，为相对定位，到达后就变为固定定位，但只局限在父元素在视口内，如果父元素都滚走了，也会一同滚走（当然此时为相对定位）

2 绝对定位元素居中的两种方式

1 flex-grow

  1. 设置放大比例，默认为 0，不能为负
  2. 当所有项目宽度之和小于容器宽度，容器有 剩余空间 = 容器宽度 - 项目宽度之和
  3. 计算所有项目的 flex-grow 之和为 s
    1. s < 1，说明放大后不会用掉所有剩余空间，则每个项目的放大尺寸 = 每个项目的 flex-grow ✖ 剩余空间
    2. s ≥ 1，说明放大后会用掉所有剩余空间，则每个项目的放大尺寸 = （每个项目的 flex-grow ➗ s）✖ 剩余空间

2 flex-shrink

  1. 设置缩小比例，默认为 1，不能为负
  2. 当所有项目宽度之和大于容器宽度，容器不足，溢出空间 = 项目宽度之和 - 容器宽度
  3. 计算所有项目的 flex-shrink 之和为 s
    1. s < 1，说明缩小后不能释放所有溢出空间，则每个项目的缩小尺寸 = 每个项目的 flex-shrink ✖ 溢出空间
    2. s ≥ 1，说明缩小后能释放所有溢出空间，则每个项目的放大尺寸 = （每个项目的 flex-shrink ➗ s）✖ 溢出空间

六 移动端

6.1 概念

  1. 屏幕尺寸：指的是屏幕对角线的长度，一般用英寸度量
  2. 屏幕分辨率：
    1. 物理分辨率（Physical resolution）
      • 硬件分辨率，出厂时就固定的，不能改变的
      • 屏幕实际的像素点数
    2. 逻辑分辨率（Logical resolution）
      • 由软件（驱动）决定的，可以缩放调节的分辨率
      • 以一个 1920 * 1080 物理分辨率为例，当缩放 150% 时，分辨率会减小，因为每个像素变大了，一行的像素自然减少了，新的分辨率 = (1920/1.5) * (1080/1.5)
  3. 设备像素比率（Device Pixel Ratio，DPR）：
    • DPR = 物理分辨率 / 逻辑分辨率
    • 高分屏，指 DPR > 1 的屏幕
  4. 布局
    1. 百分比布局（流式布局）：宽度百分比自适应，高度固定，是以前的一种布局方案
    2. 弹性布局
    3. 栅格布局

6.2 支持高分屏

  1. iphone 6 物理分辨率 750 * 1334，逻辑分辨率 375 * 667，DPR = 2

  2. 写代码针对的是逻辑分辨率，一个 375 的图片放在 750 的设备上，如果想要占满屏幕需要放大，而放大会导致失真，所以通常会提供一个 ”二倍图“

6.3 分辨率转换

  1. 相同 DPR 的设备分辨率未必相同，而提供的图片是固定的尺寸，不同的设备可能会对图片进行缩放，如果都选择一种尺寸，缩放太多会浪费性能，所以多提供不同尺寸的图片，但不同的分辨率太多，我们无法提供所有图片，所以通常提供小中大三种尺寸供选择

1 移动适配的步骤

  2. 动态计算 rem 值
    1. 利用 postcss-pxtorem
    2. 利用 less、sass、stylus 的计算能力

一 概述

二 Value Processing

  1. First, all the declared values applied to an element are collected, for each property on each element. There may be zero or many declared values applied to the element
  2. Cascading yields the cascaded value. There is at most one cascaded value per property per element
  3. Defaulting yields the specified value. Every element has exactly one specified value per property
  4. Resolving value dependencies yields the computed value. Every element has exactly one computed value per property
  5. Formatting the document yields the used value. An element only has a used value for a given property if that property applies to the element
  6. Finally, the used value is transformed to the actual value based on constraints of the display environment. As with the used value, there may or may not be an actual value for a given property on an element

2.1 Cascade

  1. Process of combining different stylesheets and resolving conflicts between different CSS rules and declarations, when more than one rule applies to a certain element.
  2. The cascade takes an unordered list of declared values for a given property on a given element, sorts them by their declaration’s precedence, and outputs a single cascaded value
  3. Only use !important as a last resource. It’s better to use correct specificities — more maintainable code!
  4. 选择器权重计算
    1. 判断选择器能否直接选中标签，不能则说明是继承，优先级最低
      • 全部是继承的时候，计算其父元素的权重
      • 一直重复，直到有结果
    2. 计算所有选择器的权重之和（每位依次对应，不会进位），然后从左到右比较每一位的大小，有结果则不再继续比较下一位

2.2 Specified value & inheritance

  1. The specified value is the value of a given property that the style sheet authors intended for that element

  2. In many cases, the specified value is the cascaded value. However, if there is no cascaded value at all, the specified value must be found some other way

    1. Inherited properties draw their defaults from their parent element through inheritance
    2. all other properties take their initial value

  3. Authors can explicitly request inheritance or initialization via the inherit and initial keywords

    值 说明
    inherit 显式开启继承（可以让属性强制继承）
    initial 设置 CSS 属性为它的默认值
    unset 如果该属性可继承，等同于 inherit，否则等同于 initial
    revert 只有很少的浏览器支持

  4. Inheritance

    1. Inheritance propagates property values from parent elements to their children.
    2. The inherited value of a property on an element is the computed value of the property on the element’s parent element. For the root element, which has no parent element, the inherited value is the initial value of the property
    3. inherited properties
      • font-*， color，line-height
      • list-*
      • text-*，letter-spacing，word-spacing
      • visibility，cursor
      • ……

2.3 Computed value

  1. Percentages
    • Percentages are measured relative to their parent’s font-size, if used to specify font-size
    • Percentages are measured relative to their parent’s width, if used to specify lengths
  2. em
    • em are measured relative to their parent font-size, if used to specify font-size
    • em are measured relative to the current font-size, if used to specify lengths
  3. rem are always measured relative to the document’s root font-size
  4. vh and vw are simply percentage measurements of the viewport’s height and width

三 Visual formatting model

3.1 概述

  1. Visual formatting model
    1. Algorithm that calculates boxes and determines the layout of theses boxes, for each element in the render tree, in order to determine the final layout of the page.
    2. 如何将文档显示在视觉媒体（屏幕, 投影仪…）上
  2. 在 Visual formatting model 中，文档树中的每个元素都会根据 “盒子模型” 转换为 0 或多个盒子，这些盒子的布局由以下因素决定：
    1. Box dimensions （the box model）
    2. Box type （inline, block, inline-block, …）
    3. positioning scheme（normal flow, float, and absolute positioning）
    4. Stacking contexts
    5. Other elements in the render tree
    6. External information (e.g., viewport size, intrinsic dimensions of images, etc.)
  3. 通常，盒子会创建一个包含其后代元素的包含块（containing block），视觉格式化模型会根据盒子的包含块的边界来渲染盒子，但盒子并不由包含块所限制，当盒子的布局跑到包含块的外面时称为溢出（overflow）

3.2 Box dimensions

1 Box model

  1. Each box has a content area (which contains its content—text, descendant boxes, an image or other replaced element content)，and optional surrounding padding, border, and margin areas. the size of each area is specified by corresponding properties, and can be zero (or in the case of margins, negative)
  2. The margin, border, and padding can be broken down into top, right, bottom, and left segments, each of which can be controlled independently by its corresponding property
  3. The perimeter of each of the four areas (content, padding, border, and margin) is called an edge, and each edge can be broken down into a top, right, bottom, and left side. Thus each box has four edges each composed of four sides:
    1. content edge or inner edge
      • The content edge surrounds the rectangle given by the width and height of the box, which often depend on the element’s content and/or its containing block size.
      • The four sides of the content edge together define the box’s content box
    2. padding edge
      • The padding edge surrounds the box’s padding. If the padding has zero width on a given side, the padding edge coincides with the content edge on that side
      • The four sides of the padding edge together define the box’s padding box, which contains both the content and padding areas
    3. border edge
      • The border edge surrounds the box’s border. If the border has zero width on a given side, the border edge coincides with the padding edge on that side
      • The four sides of the border edge together define the box’s border box, which contains the box’s content, padding, and border areas
    4. margin edge or outer edge
      • The margin edge surrounds the box’s margin. If the margin has zero width on a given side, the margin edge coincides with the border edge on that side
      • The four sides of the margin edge together define the box’s margin box, which contains the all of the box’s content, padding, border, and margin areas
  4. The background of the content, padding, and border areas of a box is specified by its background property
  5. The border area can additionally be painted with a border style using the border property

2 margin collapse

  1. In CSS, the adjoining margins of two or more boxes (which might or might not be siblings) can combine to form a single margin. Margins that combine this way are said to collapse, and the resulting combined margin is called a collapsed margin.

  2. Horizontal margins never collapse

  3. Adjoining vertical margins collapse, except:

    1. Margins of the root element’s box do not collapse.
    2. If the top and bottom margins of an element with clearance are adjoining, its margins collapse with the adjoining margins of following siblings but that resulting margin does not collapse with the bottom margin of the parent block.
    3. If the top margin of a box with non-zero computed ‘min-height’ and ‘auto’ computed ‘height’ collapses with the bottom margin of its last in-flow child, then the child’s bottom margin does not collapse with the parent’s bottom margin.

  4. Two margins are adjoining if and only if:

    1. both belong to in-flow block-level boxes that participate in the same BFC
    2. no line boxes, no clearance, no padding and no border separate them
    3. both belong to vertically-adjacent box edges
      1. top margin of a box and top margin of its first in-flow child
        • in-flow：在 normal flow 中，不是浮动、绝对定位或根元素
        • 结果：collapsed margin 相当于作用在父元素上
      2. bottom margin of box and top margin of its next in-flow following sibling
      3. bottom margin of a last in-flow child and bottom margin of its parent if the parent has auto computed height
      4. top and bottom margins of a box that does not establish a new BFC and that has zero computed min-height, zero or auto computed height, and no in-flow children

  5. 解决

    1. 父元素设置为块状格式化上下文元素
      1. 给父元素设置 overflow: hidden
      2. 父元素转为行内块元素
      3. 父元素设置浮动
    2. 父元素设置 border-top/border-bottom 值
    3. 父元素设置 padding-top/padding-bottom 值
    4. 父元素和子元素之间添加行内元素进行分隔

  6. 计算规则

    1. 如果合并外边距皆为正，取最大值
    2. 如果合并外边距皆为负，取最小值（即绝对值最大值）
    3. 如果合并外边距有正有负，取最大正外边距与最小负外边距之和

5 Calculating widths and margins

  1. Inline non-replaced elements

    • The width property does not apply.
    • A computed value of ‘auto’ for margin-left or margin-right becomes a used value of ‘0’

  2. Inline replaced elements

    • A computed value of ‘auto’ for margin-left or margin-right becomes a used value of ‘0’
    • width 有许多情况

  3. Block-level, non-replaced elements in normal flow

    ‘margin-left’ + ‘border-left-width’ + ‘padding-left’ + ‘width’ + ‘padding-right’ + ‘border-right-width’ + ‘margin-right’ = width of containing block

    • margin 可以为负，border 和 padding 不能为负
    • margin 和 width 可以为 auto
      1. 零个 auto：margin-right 被重置为 auto
      2. 一个 auto：auto 计算结果 = 满足总宽度所需的值
      3. 两个 auto：
        1. ml、mr 为 auto，width 为定值： ml 和 mr 平分剩余空间，即元素水平居中
        2. width 占一个 auto：width 尽可能大，为 auto 的外边距 = 0
      4. 三个 auto：width 尽可能大，左右外边距 = 0

  4. Floating, non-replaced elements

    • If margin-left or margin-right are computed as ‘auto’, their used value is ‘0’.
    • If width is computed as ‘auto’, the used value is the “shrink-to-fit” width.

  5. Floating, replaced elements

    • If margin-left or margin-right are computed as ‘auto’, their used value is ‘0’.
    • The used value of width is determined as for inline replaced elements.

  6. ‘Inline-block’, non-replaced elements in normal flow

    • if width is ‘auto’, the used value is the ‘shrink-to-fit’ width as for floating elements
    • If margin-left or margin-right are computed as ‘auto’, their used value is ‘0’.

6 Calculating heights and margins

  1. Inline, non-replaced elements
    • The height property does not apply. The height of the content area should be based on the font
    • The vertical padding, border and margin of an inline, non-replaced box start at the top and bottom of the content area, and has nothing to do with the line-height.
  2. Inline replaced elements
    • If margin-top, or margin-bottom are ‘auto’, their used value is 0.
    • if height and width both have computed values of ‘auto’ and the element also has an intrinsic height, then that intrinsic height is the used value of height.
    • Otherwise, if height has a computed value of ‘auto’, and the element has an intrinsic height, then that intrinsic height is the used value of height.
    • Otherwise, if height has a computed value of ‘auto’, and the element has an intrinsic ratio then the used value of height is: used width / intrinsic ratio
    • Otherwise, if height has a computed value of ‘auto’, but none of the conditions above are met, then the used value of height must be set to the height of the largest rectangle that has a 2:1 ratio, has a height not greater than 150px, and has a width not greater than the device width.
  3. Block-level, non-replaced elements in normal flow
    • if margin-top or margin-bottom are ‘auto’, their used value is 0.
    • if height is ‘auto’, the height depends on whether the element has any block-level children and whether it has padding or borders
    • Only children in the normal flow are taken into account (i.e., floating boxes and absolutely positioned boxes are ignored, and relatively positioned boxes are considered without their offset). Note that the child box may be an anonymous block box

8 Backgrounds

1 概述

  1. Each box has a background layer that may be fully transparent (the default), or filled with a color and/or one or more images
  2. The background properties specify what color（background-color）and images（background-image）to use, and how they are sized, positioned, tiled, etc.
  3. The background properties are not inherited, but the parent box’s background will shine through by default because of the initial “transparent” value on background-color

2 background-image

项 说明
Initial none
Applies to all elements
Inherited no

  1. The background of a box can have multiple layers in CSS3. The number of layers is determined by the number of comma-separated values in the background-image property. Note that a value of none still creates a layer

    1. The first image in the list is the layer closest to the user, the next one is painted behind the first, and so on.
    2. The background color, if present, is painted below all of the other layers

  2. Each of the images is sized, positioned, and tiled according to the corresponding value in the other background properties. The lists are matched up from the first value

    1. excess values at the end are not used.
    2. If a property doesn’t have enough comma-separated values to match the number of layers, the UA must calculate its used value by repeating the list of values until there are enough.

  3. A value of none counts as an image layer but draws nothing. An image that is empty (zero width or zero height), that fails to download, or that cannot be displayed (e.g., because it is not in a supported image format) likewise counts as a layer but draws nothing.

  4. When setting a background image, authors should also specify a background-color that will preserve contrast with the text for when the image is unavailable

The display property defines an element’s display type, which consists of the two basic qualities of how an element generates boxes:

  1. the inner display type, which defines (if it is a non-replaced element) the kind of formatting context it generates, dictating how its descendant boxes are laid out. (The inner display of a replaced element is outside the scope of CSS.)
  2. the outer display type, which dictates how the principal box itself participates in flow layout

2 元素类型

  1. 根据 “外在盒子” 把元素分为

    • 行内级元素 inline-level element

    • 块级元素 block-level element

  2. 根据是否具有可替换内容，把元素分为

    • 替换元素 replaced element
    • 非替换元素 non-replaced element

  3. 替换元素

    1. 元素本身没有实际内容，浏览器根据元素的类型和属性，来决定元素具体显示内容，如 img、input、textarea、iframe、video、audio、embed、canvas、object…
    2. 匿名可替换元素：通过 CSS content 属性来插入的对象被称作匿名可替换元素
    3. 所有的替换元素都是内联元素
    4. 内容的外观不受页面上的 CSS 的影响
      • 替换元素是一种外部对象，它们外观的渲染，是独立于 CSS 的
      • 样式表现在css作用域之外，如果要替换元素本身的外观，需要类似appearance的属性。或者是浏览器本身暴露的接口。但是直接input[type='checkbox']{}没办法改变内间距背景色等特性
    5. 有自己的尺寸
      • 在 Web 中，很多替换元素在没有明确尺寸设定的情况下，其默认的尺寸（不包括边框）是 300 像素×150 像素，如 <video> <iframe> <canvas>
      • 少部分替换元素为 0 像素，如<img>
      • 表单元素的替换元素的尺寸则和浏览器有关，没有明显的规律
    6. 在很多 CSS 属性上有自己的一套表现规则
      • 如 vertical-align 默认值的 baseline ，但对于替换元素，默认为元素的下边缘

  4. 非替换元素

    1. 元素本身有实际内容，浏览器直接将其内容显示出来
    2. 块级元素都是不可替换元素，即可以认为替换/非替换是针对内联元素的

3 盒子类型

4 盒子的生成

1 block

  1. block-level element：块级元素（display：block、list-item、table）
  2. block-level box：块级盒子，由块级元素生成。一个块级元素至少生成一个块级盒子（list-item 会生成两个）
  3. block container box：块容器盒子，块级盒子或其它一些盒子（如 inline-block）都可以作为块容器盒子
  4. block box：一个块级盒子同时也是块容器盒子，则称其为块盒子

2 inline

  1. Inline-level element：行内级元素（display：inline、inline-block、inline-table）
  2. Inline-level box：行内级盒子，由行内级元素生成，分为：
    1. inline box
      • An inline box is one that is both inline-level and whose contents participate in its containing inline formatting context
      • A non-replaced element with a display value of inline generates an inline box
    2. atomic inline-level box
      • Inline-level boxes and they participate in their inline formatting context as a single opaque box
      • such as replaced inline-level elements, inline-block elements, and inline-table elements
  3. Line box
    1. 行盒，由一行中的所有 inline-level box 组成，用来显示一行。The rectangular area that contains the boxes that form a line is called a line box.
      1. inline box 参与形成 line box 的是其内容，如果其内部为空，就不参与形成 line box
      2. atomic inline-level box 参与形成 line box 的是其自身，内部为空也行
    2. line box 的 height：the distance between the uppermost box top and the lowermost box bottom
      1. inline box：line-height
      2. atomic inline-level box：margin-box
    3. 空白幽灵节点 strut
      1. 每个 line box 起始位置都有一个具有该元素的字体和行高属性的 0 宽度的 inline-level box，叫做 strut（支柱）
      2. 永远透明，无法获取，表现如同文本节点一样

3 anonymous box

  1. 在某些情况下进行视觉格式化时，需要添加一些增补性的盒子，这些盒子不能用CSS选择器选中，因此称为匿名盒子

  2. 匿名盒子由于不能被选择器选中，也就不能被样式表赋予样式，即其所有可继承的 CSS 属性值都为 inherit ，而所有不可继承的 CSS 属性值都为 initial

  3. Anonymous block boxes

    1. 在一个 block container box 中如果有一个 block-level box，则 CSS 会强制要求该 block container box 中只能有 block-level box，那么 inline-level box 就只好在外面包上一个 anonymous block box

    2. When an inline box contains an in-flow block-level box, the inline box is broken around the block-level box, splitting the inline box into two boxes (even if either side is empty), one on each side of the block-level box(es). The line boxes before the break and after the break are enclosed in anonymous block boxes, and the block-level box becomes a sibling of those anonymous boxes.

  4. Anonymous inline boxex

    1. Any text that is directly contained inside a block container element (not inside an inline element) must be treated as an anonymous inline element

    2. the <p> generates a block box, with several inline boxes inside it. The box for “emphasized” is an inline box generated by an inline element (<em>), but the other boxes (“Some” and “text”) are inline boxes generated by a block-level element (<p>). The latter are called anonymous inline boxes, because they do not have an associated inline-level element.

1 概述

  1. In CSS 2, a box may be laid out according to three positioning schemes

      1. block formatting of block-level box
      2. inline formatting of inline-level box
      3. relative positioning of block-level and inline-level box
    2. Float
    3. Absolute positioning

  2. in-flow：Normal flow 中的元素，或者非 out of flow 元素

  3. out of flow

    1. root element
    2. float
    3. absolute positioning

3 Normal flow

1 概述

  1. Boxes in the normal flow belong to a formatting context, which in CSS 2.2 may be table, block or inline.

    1. block-level boxes participate in a BFC（block formatting context）
    2. inline-level boxex participate in an IFC（inline formatting context）

  2. In future levels of CSS, other types of formatting context will be introduced

2 BFC

1 概述
元素 说明
根元素 <html>
浮动元素 float: left / right
绝对定位元素 position: absolute / fixed
overflow 不为 visible overflow: hidden / auto / scroll
display display: inline-block / table-cell / table-caption / flex / inline-flex
  1. Floats, absolutely positioned elements, block containers (such as inline-blocks, table-cells, and table-captions) that are not block boxes, and block boxes with ‘overflow’ other than ‘visible’ (except when that value has been propagated to the viewport) establish new block formatting contexts for their contents
  2. In a block formatting context
    1. boxes are laid out one after the other, vertically, beginning at the top of a containing block.
    2. The vertical distance between two sibling boxes is determined by the margin properties.
    3. each box’s left outer edge touches the left edge of the containing block (for right-to-left formatting, right edges touch). This is true even in the presence of floats (although a box’s line boxes may shrink due to the floats), unless the box establishes a new block formatting context
2 应用
  1. 清除浮动：
    1. BFC 盒子会默认包裹住内部子元素（不管是标准流子元素还是浮动子元素）
    2. 由于 BFC 容器计算高度会计算浮动元素，所以只需父元素触发 BFC 即可清除浮动
  2. 两栏自适应布局
    1. 左边元素浮动，右边元素触发 BFC即可，因为 BFC 的区域不会与 float box 重叠，如图文两栏布局
    2. 图文排列时，图片浮动，就形成文字环绕
    3. 如果想让图片在左，文字在右的两列布局，只需文字容器触发 BFC 即可
  3. 解决相邻元素 margin 合并
    1. 由于 BFC 与外界元素不会相互影响，所以只需相邻元素不在同一个 BFC 容器中即可
    2. BFC 盒子本身与子元素之间不存在 margin 塌陷

3 IFC

1 概述
  1. An inline formatting context is established by a block container box that contains no block-level boxes
  2. In an inline formatting context
    1. boxes are laid out horizontally, one after the other, beginning at the top of a containing block.Horizontal margins, borders, and padding are respected between these boxes.
    2. The boxes may be aligned vertically in different ways: baseline、top….
2 术语


  1. content area
    1. 文字内容区域可以理解为选中文字时的区域
      1. content area 高度由 font-size 和 font-family 决定
      2. font-family 为 simsun(宋体) 时，content area = font-size
    2. 图片这样的替换元素，内容区域可以看成元素自身
  2. x-height：小写字母 x 的高度，即基线与中线的距离，CSS 中单位为 ex
  3. line-height：行高，即两基线之间的距离，等价于 font-size + 行间距
  4. 基线
    1. 文本内联元素的基线就是小写 x 的下边缘
    2. 类似图片这样的元素的基线就是其下边缘
    3. inline-block 元素
      1. 如果该元素中有内联元素，基线就是最后一行内联元素的基线
      2. 如果该元素内没有内联元素或者 overflow 不是 visible，其基线就是 margin 的底边缘
3 line-height
1 font-size

  1. line-height < font-size：行间距为负，会发生重叠

  2. line-height = font-size：行间距为 0，刚好不重叠所以最好设置 line-height > font-size

2 height（包含块的高度）

  1. line-height < height：此时文字在包含块中偏上的位置

  2. line-height = height：此时文字在包含块中垂直居中

  3. line-height > height：此时文字在包含块中偏下的位置，如果由多行文本，则会重叠

5 现象

  1. non-replace inline 元素形成 line box 的是内容，内部为空，没有形成 line box，所以 div 的内部 span 高度为 0，则 div 的高度也为 0

  2. atomic inline-level 元素形成 line box 的是元素本身，内部为空也能形成 line box，也就有了 幽灵节点 strut，所以虽然 span 高度为 0，但幽灵节点 strut 是有行高的，所以 div 的高度不为 0

  3. 图片底部有空隙

    1. img 形成了 line box，所以出现了 strut，vertical-align 默认为 base-line 对齐，strut 的 base-line 为 x 的下边缘，img 的 base-line 为底部
    2. 前边添加文字，现象一样

    1. 解决方式
      1. img 块级化，vertical-align 对块级元素无效（且没有了行框盒子，也就没有了 strut）
      2. 修改 vertical-align 为其他值比如 bottom top middle ...
      3. 减小 line-height 的值，也就是减小行间距，会造成文字向下移动

  4. 单行文本垂直居中

    1. 如果包含块没有设置高度，则只要设置 line-height 即可
    2. 如果包含块设置了高度，则设置 line-height = height

  5. 多行文本垂直居中

    1. 与单行文本居中，原理一样，使用 span 包含多行文本，将此 span 看成单行文本
      1. 设置包含块 line-height = height
      2. span 设置
        1. display: inline-block
        2. vertical-align: middle
          • 为了让 span 在 div 中居中
        3. 一个合适的 line-height
          • 作用于 span 内部文本的显示
    2. 为何 span 要设置 display: inline-block ?
      • span的display为默认的inline时，形成行框盒子的是span的内容，由于div的line-height会作用于行框盒子的strut，且div的line-height大于span的line-height，所以行框应用的是div的line-height，也就是说span的内容应用的是div的line-height，就会导致文字行距很大
      • span的display设置为line-block时，形成行框盒子的是span本身，span的内部还会形成一个行框盒子，这时span设置的line-height就会对内部的行框盒子起作用，即span应用的是div的line-height，sanp的内容应用的是span自身的line-height，所以文字就能正常显示

1 float

项 说明

Inherited no
Value none | left | right
  1. 早期浮动是为了实现图文环绕效果
  2. 现在多用于网页布局
    1. 让块级盒子排成一行，即垂直布局的盒子变成水平布局
    2. 盒子左右对齐

2 浮动的特性

  1. 浮动会使元素脱离标准流，漂浮在标准流上方且不再占据标准流位置，直到边缘触及到包含块或其它浮动元素的边缘
    1. 浮动元素不能超出包含块
    2. 浮动元素之间不会层叠
    3. 浮动元素的顶端不能超过父元素的顶端，也不能超过之前所有浮动元素的顶端
    4. 左浮动从右下 → 右上 → 左上，右浮动从左下 → 左上 → 右上，左浮动只能挡住左浮动，右浮动只能挡住右浮动
  2. 浮动元素不会与行内级元素层叠
  3. If there is a line box, the outer top of the floated box is aligned with the top of the current line box.
  4. 浮动会影响父元素及其后面的兄弟元素
  5. 当一个元素浮动时
    1. 其上一行
      1. 是行内级元素，且空间
        • 能够放下浮动元素，则浮动元素会在上一行浮动直到边缘触及到包含块或另一个浮动元素
        • 放不下浮动元素，则浮动元素在当前行浮动
      2. 是块级元素
        • 块级元素独占父元素一行，即上一行没有浮动元素的位置
        • 浮动元素在当前行浮动
    2. 其下一行
      1. 是行内级元素
        • 会在合适的位置环绕浮动元素
      2. 是块级元素
        • 会无视浮动元素的存在而上移

    3. both：要求设置 clear 的元素的顶部低于之前生成的所有浮动元素的底部
2 清除浮动的原因
  1. 防止浮动元素引起的父级高度塌陷
    1. 由于浮动元素会影响后边正常文档流中的元素，所以通常会把浮动元素放在一个处于正常文档流的父元素中
    2. 父元素通常不会固定高度，而是根据子元素高度来撑开高度，而浮动元素脱离文档流，不占据文档流的位置，就会导致父元素高度塌陷
  2. 防止浮动元素对后面兄弟级元素的布局影响
    1. 只需要给浮动元素的下一个兄弟元素设置清除浮动即可，后面所有元素布局都会恢复
    2. 浮动元素和清除浮动的兄弟元素之间的 margin 会失效

1 粘性定位
  1. sticky会产生动态效果，很像relative和fixed的结合：一些时候是relative定位（定位基点是自身默认位置），另一些时候自动变成fixed定位（定位基点是视口）
  2. sticky生效的前提是，必须搭配top、bottom、left、right这四个属性中的一个或多个一起使用（看滚动方向），不能省略，否则等同于relative定位，不产生”动态固定”的效果
  3. 不能有任何祖先元素设置overflow:hidden，否则没有粘滞效果。因为改变了滚动容器（即使没有出现滚动条）
  4. 父元素的高度不能低于sticky元素的高度
  5. sticky元素仅在其父元素内生效，不会超出父元素，滚动时会随着父元素一起消失
  6. 当页面滚动，父元素开始脱离视口时（即部分不可见），只要与sticky元素的距离达到生效门槛（设置的top/bottom/left/right），relative定位自动切换为fixed定位，等到父元素完全脱离视口时，fixed定位恢复成relative定位
  7. 同一个父容器中的sticky元素，如果定位值相等，则会重叠；如果属于不同父元素，则会随着父元素不再完整占据sticky元素的固定区域以后，再由其他父元素的sticky子元素占据固定位置
  8. 我的理解就是设置了粘性定位的元素，没到达所设置的偏移量时，为相对定位，到达后就变为固定定位，但只局限在父元素在视口内，如果父元素都滚走了，也会一同滚走（当然此时为相对定位）

3.5 Stacking context

1 概述

  1. The order in which the rendering tree is painted onto the canvas is described in terms of stacking contexts. Stacking contexts can contain further stacking contexts. A stacking context is atomic from the point of view of its parent stacking context; boxes in other stacking contexts may not come between any of its boxes
  2. Each box belongs to one stacking context.
  3. Each positioned box in a given stacking context has an integer stack level, which is its position on the z-axis relative other stack levels within the same stacking context.
    1. Boxes with greater stack levels are always formatted in front of boxes with lower stack levels. Boxes may have negative stack levels.
    2. Boxes with the same stack level in a stacking context are stacked back-to-front according to document tree order.
  4. stacking context：一个概念，可以想象成 z 轴上的一个栈空间
  5. stacking level：同一个 stacking context 中的元素在 z 轴栈空间的位置

2 创建 stacking context

  1. The root element forms the root stacking context.
  2. 定位元素（position 非 static）且其 z-index 不是 auto，会创建 stacking context
  3. other properties
    1. 元素为 flex 布局元素（即父元素 display:flex / inline-flex），同时 z-index 值不是 auto。
    2. 元素的 opacity 值不是 1。
    3. 元素的 transform 值不是 none。
    4. 元素 mix-blend-mode 值不是 normal。
    5. 元素的 filter 值不是 none。
    6. 元素的 isolation 值是 isolate。
    7. 元素的 will-change 属性值为上面 2～6 的任意一个（如willchange:opacity、will-chang:transform 等）。
    8. 元素的-webkit-overflow-scrolling 设为 touch。

3 stacking level

  1. Within each stacking context, the following layers are painted in back-to-front order:
    1. the background and borders of the element forming the stacking context.
    2. the child stacking contexts with negative stack levels (most negative first).
    3. the in-flow, non-inline-level, non-positioned descendants.
    4. the non-positioned floats.
    5. the in-flow, inline-level, non-positioned descendants, including inline tables and inline blocks.
    6. the child stacking contexts with stack level 0 and the positioned descendants with stack level 0.
      1. stacking context 不依赖 z-index 的 stacking context（即元素创建 stacking context 时不是用的z-index）
      2. stacking context 中 z-index 值为 0 的 stacking context 元素
      3. stacking context 中 z-index 值为 auto 的普通定位元素
        1. 元素一旦成为定位元素，其 z-index 就会自动生效，默认为auto
        2. 定位元素的 z-index 值为数值时才是 stacking context，所以此时是普通定位元素，并非 stacking context
    7. the child stacking contexts with positive stack levels (least positive first).
  2. 比较步骤
    1. 首先查看两个元素是否在同一个 stacking context
      1. 否，则查看它们所在的 stacking context 是否在同一个 stacking context，如果还不是，继续往上…直到两个stacking context 在同一个 stacking context 中，最终肯定在同一个 stacking context，因为有 root stacking context。
      2. 是，则根据它们的 stacking level 找到位置。
    2. 如果在 stacking level 中位置相同
      1. 如果明显的 stacking level 标识的，如生效的 z-index，则谁大谁在上
      2. 否则，则后来居上（即在 DOM 流中处于后面的元素会覆盖前面的元素）

6.1 What is responsive design

  1. Design technique to make a webpage adjust its layout and visual style to any possible screen size (window or viewport size)
  2. In practice, this means that responsive design makes websites usable on all devices, such as desktop computers, tablets, and mobile phones.
  3. It’s a set of practices, not a separate technology. It’s all just CSS!

6.2 Responsive design ingredients

6.3 两种开发方式

6.4 移动端

1 基本概念

1 像素

  1. 屏幕就是一块玻璃, 里面有很多的发光二极管 (灯泡). 3 个灯泡 (一个亮红色，一个亮绿色, 一个亮蓝色 RGB 原色) 组成一个最小的单元叫物理像素
  2. 一个物理像素负责显示 1 个颜色 (由 RGB 组合而来)
  3. 屏幕尺寸：指的是屏幕对角线的长度，一般用英寸度量（inch）
  4. 像素密度：PPI（pixel per inch）
    • 一英寸有多少像素，PPI 越高，画面的细节就越丰富，因为单位面积的像素数量更多
    • 常说的视网膜屏幕（Retina），就是指 PPI 较普通屏幕要高
    • $PPI = \frac{\sqrt(横向像素^2 + 纵向像素^2)}{屏幕尺寸}$

2 分辨率

  1. 物理分辨率（Physical Resolution）

    • 设备屏幕实际拥有多少物理像素
    • 硬件分辨率，出厂时就固定的，不能改变的
    • 如 1366*768 表示水平方向有 1366 个像素，垂直方向有 768 个像素

  2. 逻辑分辨率（Logical Resolution）

    • 没有视网膜屏幕（Retina）之前，是没有逻辑像素的，在 CSS 中写多少像素，就占用设备多少个物理像素。但出现了 Retina 之后，屏幕 PPI 越来越大，屏幕上分布的像素点越来越多，如果仍用 CSS 的 1px 占用设备的 1 个物理像素，若在一个 低 PPI 的屏幕上正常显示，则在高 PPI 屏幕上就会显得很小。为了解决这个问题，就在物理像素之上抽取了一层，以固定数量的物理像素来表示一个逻辑像素，重新构成设备显示的像素点

    • CSS 中的像素指的就是逻辑像素

  3. 设备像素比率（Device Pixel Ratio，DPR）：以多少个物理像素表示一个逻辑像素

    • DPR = 物理分辨率 / 逻辑分辨率
    • 高分屏（High Density）：指 DPR > 1 的屏幕

2 Responsive Images

1 概述

  1. The goal of responsive images is to serve the right image to the right screen size and device, in order to avoid downloading unnecessary large images on smaller screens
  2. Resolution Switching 和 Density Switching（特殊的 Resolution Switching）：使用的是同一个图片的不同分辨率
  3. Art Direction：使用的是不同的图片，但其重点显示的内容相同

2 <img>

  1. H5 中 img 有两个新属性 srcset 和 sizes，作用

    • 控制响应式图片，根据屏幕大小来选择显示不同图片
    • 根据屏幕的不同（视网膜屏幕还是普通屏），展示不同质量的图片，合理控制下载资源，以及带给用户高质量享受

  2. srcset

    1. One or more strings separated by commas, indicating possible image sources for the UA to use. Each string is composed of:

      1. A URL to an image
      2. Optionally, whitespace followed by one of:
        • A pixel density descriptor (a positive floating point number directly followed by x).
        • A width descriptor (a positive integer directly followed by w). The width descriptor is divided by the source size given in the sizes attribute to calculate the effective pixel density.
      3. If no descriptor is specified, the source is assigned the default descriptor of 1x.

    2. 例

  3. sizes

    1. One or more strings separated by commas, indicating a set of source sizes. Each source size consists of:
      1. A media condition. This must be omitted for the last item in the list.
      2. A source size value
    2. 只有当设置了srcset，且单位为 w 时，sizes 的设置才会起效

  4. 图片的选择

    1. x 图片的选择：根据对应规则即可
    2. w 图片的选择
      1. 获取 sizes 的值，缺省为 100vw，sizes 的值是逻辑像素，而 srcset 中的 w 代表的是物理像素，所以需要计算 sizes 的物理像素（sizes * DPR）
      2. 用 sizes 的物理像素去匹配 srcset 中的 w，选择合适的图片，如果超过最大的 w，则选择最大的那个

  1. 相同 DPR 的设备物理分辨率未必相同，如果只提供一种尺寸的图片，不同的设备可能会对图片进行缩放，缩放太多会浪费性能，所以多提供不同尺寸的图片，但不同的分辨率太多，我们无法提供所有图片，所以通常提供小中大三种尺寸供选择

2 语法

1 media query

  1. The syntax of a media query consists of an optional media query modifier, an optional media type, and zero or more media features
  2. A media query is a logical expression that is either true or false
    1. A media query is true if the media type, if specified, matches the media type of the device where the user agent is running, and the media condition is true
    2. If the media query list is empty, it evaluates to true.

二 Transitions

2.1 概述

  1. Normally when the value of a CSS property changes, the rendered result is instantly updated, with the affected elements immediately changing from the old property value to the new property value.
  2. This section describes a way to specify transitions using new CSS properties. These properties are used to animate smoothly from the old state to the new state over time.

2.1 transition

1 transition-property

specify the name of the CSS property to which the transition is applied，对于简写属性，其完整版属性都应用过渡

项 说明
Initial all
Applies to all elements
Inherited no
值 说明
all（默认） 所有可过渡的属性都使用过渡（不是所有 css 属性都有过渡效果，一般来讲有中间值的属性都可以）
none 没有属性会获得过渡效果
<custom-ident> 定义应用过渡效果的 CSS 属性名称列表，属性间以逗号分隔

2 transition-duration

This property specifies how long the transition from the old value to the new value should take

  • 值不能为负
  • 一个值时，所有属性使用同样的时间
  • 多个值时，如果属性数量大于值数量，超出的属性再从第一个时间开始重复使用
项 说明
Initial 0s
Applies to all elements
Inherited no
Value <time>#

3 transition-timing-function

项 说明
Initial ease
Applies to all elements
Inherited no
Value <time-function>#

1 曲线

值 说明
ease（默认） 慢 → 快 → 慢 → 非常慢（等价于 cubic-bezier(0.25, 0.1, 0.25, 1)）
linear 匀速（等价于 cubic-bezier(0, 0, 1, 1)）。
ease-in 慢 → 快（等价于 cubic-bezier(0.42, 0, 1, 1)）
ease-out 快 → 慢 （等价于 cubic-bezier(0, 0, 0.58, 1)）
ease-in-out 慢 → 快 → 慢（等价于 cubic-bezier(0.42, 0, 0.58, 1)）
cubic-bezier(n,n,n,n) 三次贝塞尔曲线

2 步进

值 说明
steps(n, start) 将过渡过程分为 n 个阶段，每个阶段从阶段开始就变化，即每个阶段一开始就是此阶段结束时的效果
steps(n, end) 将过渡过程分为 n 个阶段，每个阶段到阶段结束才变化，即每个阶段一开始是此阶段开始时的效果
step-start 同 steps(1, start)
step-end 同 steps(1, end)

4 transition-delay

defines when the transition will start

  • 必须在过渡时间之后设置
  • 默认为 0s，即立刻开始过渡
  • 值可以为负数，如-3s，表示直接显示在 3s 时的效果
  • 过渡属性数量大于时间数量时，后面的属性从第一个时间开始重复使用
项 说明
Initial 0s
Applies to all elements
Inherited no
Value <time>#

2.2 TransitionEvent

1 事件

事件 说明
transitionend 在 CSS 过渡完成后触发
transitioncancel 在 CSS 过渡取消时触发
transitionrun 在创建了 CSS 过渡（不一定要启动），将其添加到一组正在运行的过渡中时触发
transitionstart 在 CSS 过渡开始过渡时触发

2 属性

属性 说明
propertyName 过渡的属性名，只读
elapsedTime 过渡已经进行的时间，只读
pseudoElement 过渡的伪元素名
isTrusted true 用户触发，false 脚本触发

  2. 变形操作一次定义多个，使用空格隔开，每个相对于上一个结果变化，有顺序，旋转建议放在后边，因为会改变坐标轴方向

3.3 透视

1 概述

  1. 现实中，眼镜看物体近大远小，近实远虚。默认情况下，电脑屏幕是平面，无法观察远近效果
  2. perspective 属性指定了观察者与 z=0 平面的距离（视距），使具有三维位置变换的元素产生透视效果，透视点是在浏览器的前方，即眼睛距离屏幕的垂直距离
  3. perspective 透视值体现了离物体的远近，建议数值 800 ~ 1200px，如果数值太小，说明眼睛离屏幕太近，大小变化太明显，如果数值太大，大小变化不明显
  4. perspective 属性只是增加了近大远小，近实远虚的视觉效果，不能呈现立体图形，想要实现 3D 需要 transform-style

2 设置透视

项 说明
Name perspective
Initial none
Applies to transformable elements
Inherited no
Value `none
方式 名称 说明
perspective: <length> 舞台透视 定义在父级，影响子元素且子元素透视效果不一，就好像在一个地方去观察一排杯子一样
transform: perspective( <length> ) 单独透视 定义在当前元素上（与其他变形一起且在其它变形之前），就好像分别在每个杯子前面去观察

3 透视基点

Applies to transformable elements
Inherited no
Percentages refer to the size of the reference box
Value <position>

3.4 3D 呈现

项 说明
Name transform-style
Initial flat
Applies to transformable elements
Inherited no
Value flat | preserve-3d
值 说明
flat（默认） 2D 效果
preserve-3d 开启 3D 立体环境，定义在父级，使子元素处于真正的 3D 空间

3.5 隐藏背面

项 说明
Name backface-visibility
Initial visible
Applies to transformable elements
Inherited no
Value visible | hidden
  1. backface-visibility 属性用来定义当一个元素背面朝向屏幕时是否可见
  2. 就好像一个魔方
    • visible：可以看见里边那个面的背面，像眼睛能透视一样
    • hidden：不可见
  3. The visibility of an element with backface-visibility: hidden is determined as follows:
    1. Compute the element’s accumulated 3D transformation matrix
    2. If the component of the matrix in row 3, column 3 is negative, then the element should be hidden. Otherwise it is visible.

四 动画

4.1 概述

  1. 过渡和动画
    • 过渡：两个状态之间的变化过程
    • 动画：实现多个状态间的变化过程，动画过程可控（重复播放、最终画面、是否暂停）
  2. 动画的本质是快速切换大量图片时在人脑中形成的具有连续性的画面
  3. 构成动画的最小单元：帧或动画帧
  4. 无缝动画：将初始状态复制一份放在最后

4.2 定义动画

关键帧 说明
from 相当于 0%，即动画的开始，未设置则使用元素现有状态作为动画开始状态
to 相当于 100%，即动画的结束，未设置则使用元素现有状态作为动画结束状态
<percentage> 表示此关键帧处于动画的哪个阶段，占动画时长的百分比

  1. 定义使用的关键帧

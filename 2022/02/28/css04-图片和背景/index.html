04-图片和背景

发表于 2022-02-28 分类于 CS ， CSS

一 概述

1.1 图片类型

  1. 位图
    1. 来自相机、扫描仪或由软件生成
    2. 由像素组成
    3. 格式：jpg、png、gif、webp
    4. 放大会失真
  2. 矢量图
    1. 来自数学工具软件（如 Illustrator）
    2. 由数学矢量（直线、曲线…）构成
    3. 格式：svg
    4. 缩放不失真，可用于图标、logo、简单图形或背景

1.2 位图格式

格式 colors（色深） transparency（透明度） animation（动画）
JPEG 16M

  1. 如何转换成 webp

  2. 使用

二 background

1 background-color

值 说明
transparent（默认） 透明
currentColor 当前元素的字体颜色
<color> 设置颜色

2 background-image

值 说明
none（默认） 无背景图
url url(地址不提倡加引号)
<gradient> 渐变
  • img 图片和背景图片
    • img 图片可单独使用，有默认宽高
    • 背景图片是装饰，不能撑开盒子

3 background-repeat

  • css3 定义时使用两个值，如果只设置一个，也会被解析成两个
单值 说明 双值
repeat（默认） 平铺 repeat repeat
repeat-x x 轴平铺 repeat no-repeat
repeat-y y 轴平铺 no-repeat repeat
no-repeat 不平铺 no-repeat no-repeat
space 两端对齐平铺，多出来的空间用空白代替 space space
round 两端对齐平铺，多出来空间通过自身的拉伸来填充 round round

4 background-position

  • x轴 y轴，如果只设置一个值，另一个默认居中
值 说明
关键字 left right top bottom center
<length> 长度 px em ...（水平向右垂直向下为正）
<percentage> 百分比

5 background-size

  • 宽 高，如果只设置一个值，另一个值为 auto，保持宽高比
值 说明
auto（默认） 以背景图片的本身大小显示
cover 等比例缩放背景图片以完全覆盖背景区，背景区没有空白，背景图片可能部分看不见
contain 等比例缩放背景图片以完全装入背景区，背景区可能部分空白
<length> 长度
<percentage> 背景图片相对当前盒子宽高的百分比

6 background-attachment

  • 决定背景图像的位置是在视口内固定，还是随着包含它的区块滚动。
值 说明 元素滚动条 页面滚动条
scroll（默认） 背景相对于元素本身固定，融为一体 背景不动 背景会动
fixed 背景相对于视口固定 背景不动 背景不动
local 背景相对于元素的内容固定 背景会动 背景会动

7 background-origin

  • 决定背景图片相对于哪个盒子定位
值 说明
padding-box（默认） 相对于 padding-box 定位
content-box 相对于 content-box 定位
border-box 相对于 border-box 定位

8 background-clip

  • 设置元素的背景切割范围，相对于哪个盒子定位不变
值 说明
border-box（默认） 背景延伸至边框外沿（但是在边框下层）
content-box 背景被裁剪至内容区外沿
padding-box 背景延伸至内边距外沿

三 技巧

3.1 精灵图

1 概述

  1. 为了有效地减少服务器接受和发送请求的次数，提高页面的加载速度，将网页中的一些背景图像整合到一张大图中（精灵图），这样只需要一次请求即可
  2. CSS 精灵技术主要针对于背景图片，内容图片 img 是不需要这个技术的
    1. 不要将太大的图片加入到精灵图
    2. 精灵图没有扩展性
    3. 通常用在图标和logo上

2 使用

  1. 精确测量每个小图标的大小和位置
  2. 创建与小图标大小相同的盒子，将精灵图设置为盒子背景
  3. 给盒子指定背景图位置（背景定位基本都是负值，因为正常向右和向下为正，而这里盒子不动，移动的是图片，所以向相反方向移动）

3.2 Data URIs

  1. 可以用在任何需要图片 src 属性的地方
    1. 好处：将图片直接嵌入到网页中，在客户端完成解码，就不用进行 Http 请求了
    2. 坏处：
      1. 增加原文件大小
      2. 维护困难
      3. 移动设备使用比直接使用图片慢
  2. 格式：data:[<mini type>][;charset=<charset>][;base64],<encoded data>

3.3 图片裁切：clip-path

  1. 使用裁剪方式创建元素的可显示区域。区域内的部分显示，区域外的隐藏
  2. 搜索工具网站，复制代码..

3.4 滤镜：filter

filter 说明
blur() 模糊，取值 radius 设定高斯函数标准差，或 <number> + px，默认 0
brightness() 亮度，取值 0% ~ 100%（可超，会更亮），默认 1
contrast() 对比度，取值 0% ~ 100%（可超），默认 1
grayscale() 灰度，取值 0% ~ 100%，默认 0
saturate() 饱和度，取值 0% ~ 100%（可超），默认 1

3.5 支持高分屏

  1. 以 iphone 3和4 为例，同样尺寸的两个屏幕，一个分辨率是 480 * 320，一个分辨率是 960 * 640，把一个 320 * 320 的图片放在两个手机上水平方向都会占满，因为 iphone4 的缩放系数 scale factor = 2，这样很好，但有一个问题，这相当于把图片放大了 1 倍，而图片放大会失真，所以需要一个 640 * 640 的图片替换

  2. 术语

    1. Physical resolution：物理分辨率，屏幕实际的像素点数，iphone4 的是 960 * 640
    2. Logical resolution：逻辑分辨率（css 的计算方式），iphone4 的是 480 * 320
    3. Device Pixel Ratio（DPR）：设备像素比率 = Physical / Logical
    4. High Density：高分屏，指 DPR > 1 的屏幕

  3. 使用

3.6 分辨率转换

  1. 相同 DPR 的设备分辨率未必相同，而提供的图片是固定的尺寸，不同的设备可能会对图片进行缩放，如果都选择一种尺寸，缩放太多会浪费性能，所以多提供不同尺寸的图片，但不同的分辨率太多，我们无法提供所有图片，所以通常提供小中大三种尺寸供选择

  2. 使用

3.7 美术指导（art direction）

  1. 不同设备看同一张照片的效果不一样，所以有时想要提供一个当前图片放大局部或者裁切的效果，这就是美术指导

  2. 使用

一 字体

1.1 概述

1 字体格式

  1. TTF (TrueType Font)
    1. 由苹果和微软为 PostScript 而开发的字体格式。在 Mac 和 Windows 操作系统上，TTF 一直是最常见的格式，所有主流浏览器都支持它。然而，IE8 不支持 TTF；且 IE9 上只有被设置成 “installable” 才能支持（译注：别想了，转别的格式吧）。
    2. TTF 允许嵌入最基本的数字版权管理标志————内置标志可以告诉我们字体作者是否允许改字体在 PDF 或者网站等处使用，所以可能会有版权问题。另一个缺点是，TTF 和 OTF 字体是没压缩的，因此他们文件更大
  2. OTF (OpenType Font)
    1. 由 TTF 演化而来，是 Adobe 和微软共同努力的结果。OTF 字体包含一部分屏幕和打印机字体数据。OTF 有几个独家功能，包括支持多平台和扩展字符集。OTF 字体可以在 Mac 和 Windows 系统上使用
    2. OTF 也允许多达 65000 个字符的存储。这个额外的空间让设计师可以自由地添加附加元素，比如小帽子、老式数字体、代替的字符和其他一些以前必须作为独立字体分发的附加材料
  3. EOT (Embedded Open Type)
    1. 是微软设计用来在 Web 上使用的字体。是一个在网页上试图绕过 TTF 和 OTF 版权的方案。
    2. 你可以使用微软的工具从现有的 TTF/OTF 字体转成 EOT 字体使用，其中对字体进行压缩和裁剪使得文件体积更小。同时为了避免一些收版权保护的字体被随意复制，EOT 还集成了一些特性来阻止复制行为，以及对字体文件进行加密保护。听起来很有前途？嗯哼，可惜 EOT 格式只有 IE 支持
  4. WOFF (Web Open Font Format)
    1. 本质上是 metadata + 基于 SFNT 的字体（如 TTF、OTF 或其他开放字体格式）。该格式完全是为了 Web 而创建，由 Mozilla 基金会、微软和 Opera 软件公司合作推出。 WOFF 字体均经过 WOFF 的编码工具压缩，文件大小一般比 TTF 小 40%，加载速度更快，可以更好的嵌入网页中。metadata 允许在字体文件中包含许可数据，以解决版权问题。这是万维网联盟提倡的，所以毫无疑问的是字体格式的未来。目前主流的浏览器的新版本几乎都支持 WOFF
    2. WOFF2 是 WOFF 的下一代。 WOFF2 格式在原有的基础上提升了 30% 的压缩率。由于它还没有 WOFF 的广泛支持，所以还只是一个可展望的升级
  5. SVG (Scalable Vector Graphics font)
    1. 字体格式使用 SVG 的字体元素定义。这些字体包含作为标准 SVG 元素和属性的字形轮廓，就像它们是 SVG 映像中的单个矢量对象一样
    2. SVG 字体最大的缺点是缺少字体提示（font-hinting）。字体提示是渲染小字体时为了质量和清晰度额外嵌入的信息。同时，SVG 对文本（body text）支持并不是特别好。因为 SVG 的文本选择（text selection）目前在 Safari、Safari Mobile 和 Chrome 的一些版本上完全崩坏，所以你不能选择单个字符、单词或任何自定义选项，你只能选择整行或段落文本

2 字体族（font family）

类型 说明 使用场景 例
serif 衬线字体，边角有特殊的装饰线或衬线 报刊、书籍… Georgia、Times New Roman、宋体
sans-serif 无衬线字体，笔画粗细一致，没有边角装饰，注：’sans’ 是法语没有的意思 网页 Avenir、Arial、Futura、Helvetica、Roboto、黑体
monospace 等宽字体，每个字符所占宽度一样 代码 Consolas、Courier、Ubantu

1.2 @font face

1 作用

  1. 通常情况下，网页字体只能使用系统已安装的字体，所以定义字体时通常需要定义一种网络安全字体（几乎所有电脑都能正常显示的字体）以确保用户的网页能正常显示
  2. @font-face 允许网页开发者为其网页指定在线字体，用户可以从网上下载并使用自定义字体，从而消除对用户电脑字体的依赖
  3. 字体类型
    1. 网络安全字体：
    2. 自定义字体（先使用 @font face 注册，之后正常使用）
      1. 本地字体
      2. 字体服务器
    3. 系统字体
  4. 自定义字体或字体族
  5. 它允许网页开发者为其网页指定在线字体，可以消除

2 语法

属性 说明
font-family 自定义字体的名称，特殊符号必须使用引号
src 指定本地字体或在线字体，format('woff2') 转换字体格式，用于浏览器识别
unicode-range 可以让特定的字符或者字符片段使用特定的字体

3 font-display

值 说明
auto 默认值，不同浏览器做法不同
swap 浏览器使用备用字体，直到自定义字体下载完成，可能的问题是用户正在浏览时字体突变
fallback 如果短期内自定义字体没有下载完成，即使后来下载完了，也一直使用备用字体
optional 类似 fallback，但会在后台下载自定义字体并缓存以后使用，推荐
block 自定义字体下载完成前，会一直隐藏文字，不推荐
  1. 问题：
    1. Flash of Unstyled Text：无样式字体闪现
    2. 当使用自定义字体时，用户在字体下载完成前，有可能会先使用备用字体代替，可能会导致排版问题
  2. 解决：font-display: optional

2 font-weight

值 说明
normal（默认） 常规，相当于 400
bold 加粗，相当于 700

<number [100,900]> [100, 900]

3 font-size

值 说明
keyword xx-small, x-small, small, medium, large, x-large, xx-large, xxx-large, larger, smaller
<length> <number> + 长度单位，如 px em ex rem
<percentage> 相对父元素字体大小的百分比

4 line-height

值 说明
normal（默认） 与 font-family 有关，不同浏览器和系统字体不同，默认值就不同，所以通常需要重新设置
<number> 推荐，字体大小的倍数。
<length> <number> + 长度单位，如 px em ex rem
<percentage> 字体大小的百分比

5 font-family

  1. 渲染规则：从左往右按顺序查找，若电脑未安装该字体，则显示下一个，若都不支持，则根据操作系统显示最后字体族的默认字体
  2. 字体名称中若有多个单词，推荐用引号包裹，最后一项字体族不需要引号

2.3 溢出

1 text-overflow

when inline content overflows its end line box edge in the inline progression direction of its block container element

项 说明
Initial clip
Applies to block containers
Inherited no
值 说明
clip（默认） 将溢出的文本裁减掉
ellipsis 将溢出的文本用省略号 ... 来表示

2 overflow-wrap

原 word-wrap，指定一个不可分割的字符串过长，超过容器盒的宽时应该如何处理，只有 white-space 允许换行时才起作用

项 说明
Initial normal
Applies to non-replaced inline elements
Inherited yes
值 CJK（中日韩） non-CJK
normal（默认） 单个字符之间换行 在正常的单词结束处换行，保留完整性，可能会溢出容器
break-word 单个字符之间换行 内容将在边界内换行。如果需要，单词内部允许断行

02-CSS选择器

发表于 2022-02-28 更新于 2022-04-07 分类于 CS ， CSS

一 选择器

1.1 基础选择器

选择器 说明
html 标签选择器
#id id 选择器
.class class 选择器
* 通配选择器
S₁, S₂, S₃, ... 并集
S₁S₂S₃... 交集

1.2 关系选择器

关系选择器 说明
A E E 为 A 的后代元素
A > E E 为 A 的子元素
A ~ E E 为 A 的兄弟元素
A + E E 为 A 的下一个兄弟元素
  • 注：浏览器是从右往左读的，所以关系选择器效率不如基础选择器

1.3 属性选择器

属性选择器 说明
[attr] 带有 attr 属性的元素
[attr=value] 带有 attr 属性且值为 value 的元素
[attr^=value] 带有 attr 属性且值以 value 开头的元素
[attr$=value] 带有 attr 属性且值以 value 结尾的元素
[attr*=value] 带有 attr 属性且值包含 value 的元素

1.4 伪类选择器

1 动态伪类

选择器 说明
:link 未访问
:visited 已访问
:hover 悬停
:active 点击
:focus 获取焦点的元素

  1. 建议顺序 L V F H A

  2. hover/active 除了 a 外，还能用于其它元素

  3. 直接给 a 标签设置样式，相当于给 a 元素的所有动态伪类都设置了

  4. 如果不想让 a 标签被 tab 键选中

2 结构伪类

1 无类型

选择器 说明
:first-child 第一个子元素
:last-child 最后一个子元素
:noly-child 唯一子元素
:nth-child(表达式) 从前向后匹配符合表达式结果的子元素
:nth-last-child(表达式) 从后向前匹配符合表达式结果的子元素
  1. 子元素从 1 开始编号
  2. 通常使用交集的写法
    1. :firs-child：匹配父元素下的第一个子元素
    2. E:first-child：交集，匹配父元素下的第一个且是 E 类型的子元素。若不是第一个或第一个不是 E 类型则无法匹配

2 带类型

选择器 说明
:fitst-of-type 匹配父元素下每种类型中的第一个子元素
:last-of-type 匹配父元素下每种类型中的最后一个子元素
:noly-of-type 匹配父元素下每种类型唯一的子元素
:nth-of-type(表达式) 从前向后匹配每种类型中符合表达式结果的子元素
:nth-last-of-type(表达式) 从后向前匹配每种类型中符合表达式结果的子元素
  1. :first-of-type：匹配父元素下以类型为单位，每种类型中的第一个子元素，不要求一定是父元素的第一个子元素
  2. E:first-of-type：交集，匹配父元素下 E 类型中的第一个子元素

3 表达式

表达式示例 说明
2n，even 偶数
2n+1，2n-1，odd 奇数
-n+m 前 m 个
n+m 从第 m 个往后

3 元素状态伪类

选择器 说明
:checked 选中的 radio checkbox option
:default 默认选中的 radio checkbox option
:disabled 禁用的元素
:enabled 启用的元素
:empty 没有子元素的元素
:root <html> 元素

4 否定伪类

选择器 说明
:not(S) 非（权重由括号内选择器决定，自身0，不支持复杂选择器）

5 目标伪类

选择器 说明
:target 选中当前被激活的锚点

1.5 伪元素选择器

伪元素选择器 说明
::before 在父元素内容的最前添加一个伪元素
::after 在父元素内容的最后添加一个伪元素
::first-letter 匹配块级元素第一行的第一个字母
::first-line 匹配块级元素第一行
::selection 匹配被用户选中或者处于高亮状态的部分
  1. 伪元素：一般页面中非主体内容可以使用伪元素
    • 元素：HTML 设置的标签
    • 伪元素：由 CSS 模拟出的标签效果
  2. ::before, ::after 必须设置 content 属性才生效
  3. 伪元素默认是行内元素

二 权重

规则 粒度
!important ∞
行内样式 1, 0, 0, 0
id 0, 1, 0, 0
class，伪类，属性 0, 0, 1, 0
标签，伪元素 0, 0, 0, 1
* 0, 0, 0, 0
继承 null（继承没有权重，比 0 小）

  1. 权重的四个数有低位高位的概念

    1. 不会进位，如 0,0,0,8 + 0,0,0,7 = 0,0,0,15
    2. 从左到右依次代表：行内样式的个数、id选择器的个数、类选择器的个数、标签选择器的个数

  2. 权重的计算

    1. 判断选择器能否直接选中标签，不能则说明是继承，优先级最低
      • 全部是继承的时候，计算其父元素的权重
      • 一直重复，直到结果
    2. 计算所有选择器的权重之和，然后从左到右比较每一位的大小，有结果则不再继续
    3. 如果优先级相同，谁在后用谁的

三 继承

值 说明
inherit 显式开启继承（可以让属性强制继承）
initial 设置 CSS 属性为它的默认值
unset 如果该属性默认可继承，该值等同于 inherit，如果该属性是非继承属性，该值等同于 initial
revert 只有很少的浏览器支持

  1. 继承性：子元素继承父元素的样式

  2. CSS 提供了四个通用属性值来控制继承，元素的每个属性都可以设置这四个值

  3. CSS 继承的是计算值，而不是字面量

  4. 继承失效的情况：元素有浏览器默认样式，此时继承性仍然存在，但优先显示浏览器的默认样式（因为作用再元素自身上）

    1. <a> 的 color 会继承，但被默认样式覆盖了
    2. <h> 的 font-size 会继承，但被默认样式覆盖了
    3. <div> 的高度不能继承，宽度也不能继承，但有类似于继承的效果（块元素独占一行特性）

  5. 元素可继承属性

四 层叠

4.1 概述

概念 名称 说明
stacking context 层叠上下文 层叠上下文是一个概念，可以想象成 z 轴上的一个层叠结界
stacking level 层叠等级 决定了同一个层叠上下文中元素在 z 轴上的显示顺序
  1. 层叠特性
    1. 给同一个标签设置不同的样式：样式会层叠叠加，共同作用在标签上
    2. 给同一个标签设置相同的样式：样式会层叠覆盖，根据层叠上下文和层叠等级判断哪个样式生效
  2. 当样式冲突时，只有选择器优先级相同时，才能通过层叠性判断结果

4.2 层叠上下文

1 特性

  1. 层叠上下文的层叠水平要比普通元素高。
  2. 层叠上下文可以阻断元素的混合模式
  3. 层叠上下文可以嵌套，内部层叠上下文及其所有子元素均受制于外部的“层叠上下文”。
  4. 每个层叠上下文和兄弟元素独立，也就是说，当进行层叠变化或渲染的时候，只需要考虑后代元素。
  5. 每个层叠上下文是自成体系的，当元素发生层叠的时候，整个元素被认为是在父层叠上下文的层叠顺序中

2 创建层叠上下文

  1. 页面根元素<html>天生具有层叠上下文，称为根层叠上下文。
  2. 定位元素（position非static）的 z-index 不是auto，会创建层叠上下文
  3. 其它css3属性创建层叠上下文
    1. 元素为 flex 布局元素（即父元素 display:flex / inline-flex），同时 z-index值不是 auto。
    2. 元素的 opacity 值不是 1。
    3. 元素的 transform 值不是 none。
    4. 元素 mix-blend-mode 值不是 normal。
    5. 元素的 filter 值不是 none。
    6. 元素的 isolation 值是 isolate。
    7. 元素的 will-change 属性值为上面 2～6 的任意一个（如willchange:opacity、will-chang:transform 等）。
    8. 元素的-webkit-overflow-scrolling 设为 touch。

4.3 层叠等级

1 同一个层叠上下文中的七层

  1. 层叠上下文的背景和边框
  2. 层叠上下文中z-index值为负的层叠上下文元素
  3. 层叠上下文中块级普通元素
  4. 层叠上下文中浮动元素
  5. 层级上下文中内联普通元素
  6. 此层包含三种
    1. 层叠上下文中不依赖z-index 的层叠上下文元素（即元素创建层叠上下文时不是用的z-index）
    2. 层叠上下文中z-index值为 0 的层叠上下文元素
    3. 层叠上下文中z-index值为 auto 的普通定位元素
      1. 元素一旦成为定位元素，其z-index就会自动生效，默认为auto
      2. 定位元素的z-index值为数值时才是层叠上下文，所以此时是普通定位元素，并非层叠上下文
  7. 层叠上下文中z-index值为负的层叠上下文元素

2 比较步骤

  1. 首先查看两个元素是否在同一个层叠上下文
    1. 否，则查看它们所在的层叠上下文是否在同一个层叠上下文，如果还不是，继续往上…直到两个层叠上下文在同一个层叠上下文中，最终肯定在同一个层叠上下文，因为有根层叠上下文。
    2. 是，则根据它们的层叠水平，在层叠顺序中找到位置。
  2. 如果在层叠顺序中位置相同
    1. 如果明显的层叠水平标识的，如生效的 z-index，则谁大谁在上
    2. 如果没有明显的层叠水平标识的，则后来居上（即在 DOM 流中处于后面的元素会覆盖前面的元素）

3 z-index

  • 只能用在 定位元素 和 flex 布局中的项目
值 说明
auto（默认） 不会创建一个新的本地堆叠上下文
<integer> 创建层叠上下文，可以为负

1.5 margin

1 语法

1 计算规则

  1. 如果一侧定值，一侧 auto，则 auto 为剩余空间大小。
  2. 如果两侧均是 auto，则平分剩余空间

2 触发计算前提条件

  1. width 或 height 为 auto 时，元素是具有对应方向的自动填充特性
  2. 自动填充特性
    1. 块级元素 width 为 auto 时，水平方向会自动填充
    2. 绝对定位元素
      1. 如果设置 left 和 right 为 0，则水平方向填充特性
      2. 如果设置 top 和 bottom 为 0，则垂直方向有填充特性

3 margin 合并

1 概述

  1. 块级元素的上外边距（margin-top）与下外边距（margin-bottom）有时会合并为单个外边距
  2. 不包括浮动和绝对定位元素，尽管浮动和绝对定位可以让元素块状化

2 场景

  1. 相邻兄弟元素 margin 合并

  2. margin-top 传递

    1. 块级元素顶部线与父元素顶部线重叠，则此块级元素的 margin-top 值会传递给父元素

    2. 解决 margin-top 传递

      1. 父元素设置为块状格式化上下文元素

        1. 给父元素设置 overflow: hidden
        2. 父元素转为行内块元素
        3. 父元素设置浮动

      2. 父元素设置 border-top 值

      3. 父元素设置 padding-top 值

      4. 父元素和第一个子元素之间添加内联元素进行分隔

  3. margin-bottom 传递

    1. 块级元素底部线与父元素底部线重叠，并且父元素的高度是 auto，则此块级元素的 margin-bottom 值会传递给父元素
    2. 解决margin-bottom 传递
      1. 父元素设置为块状格式化上下文元素
      2. 父元素设置 border-bottom 值
      3. 父元素设置 padding-bottom 值
      4. 父元素和最后一个子元素之间添加内联元素进行分隔
      5. 父元素设置 height、min-height 或 max-height

  4. 空块级元素的 margin 合并

    • 解决
      1. 设置垂直方向的 border；
      2. 设置垂直方向的 padding；
      3. 里面添加内联元素（直接 Space 键空格是没用的）；
      4. 设置 height 或者 min-height。

3 计算规则

  1. 如果合并外边距皆为正，取最大值
  2. 如果合并外边距皆为负，取最小值（即绝对值最大值）
  3. 如果合并外边距有正有负，取最大正外边距与最小负外边距之和

4 margin 无效的场景

  1. 非替换内联元素的垂直方向（水平方向有效）
  2. <tr><td>标签或者 display为 table-row table-cell 的元素
  3. 绝对定位元素非定位方位的 margin 值表面“无效”

1.6 盒子宽高

1 box-sizing

值 说明
content-box（默认） width/height 针对内容盒子
border-box width/height 针对边框盒子

2 width

值 说明
auto（默认） 浏览器将会为指定的元素计算并选择一个宽度.
<length> 指定值
<percentage> 指定为包含块宽度的百分比
fill-available 元素宽度自动填满可用空间
fit-content 元素宽度包裹内容，文本内容正常换行。如浮动、绝对定位、inline-block 元素或 table 元素
min-content 元素宽度为内部元素”首选最小宽度”最大的那一行的宽度
max-content 元素宽度为内部元素宽度值最大的那个元素的宽度
  1. 首选最小宽度
    1. 指的是元素最适合的最小宽度
    2. 类似图片这样的替换元素的最小宽度就是该元素内容本身的宽度
    3. 文本
      • CJK（中日韩）可以任意断行，最小宽度就是每个字符的宽度
      • non-CJK一般在空格、短横线、问号以及其他非英文字符等断行，最小宽度由连续的英文字符单元决定
  2. fit-content 和 max-content 区别
    1. 两者的元素宽度其实都是内部元素最大的那个宽度
    2. 区别在于 fit-content 内部元素文字可以正常换行，而 max-content 内部元素文字不换行
  3. CSS 流体布局下的宽度分离原则
    1. 宽度分离原则即CSS 中的 width 属性不与影响宽度的 padding/border（有时候包括 margin）属性共存
    2. 应该width 独立占用一层标签，而padding、border、margin 利用流动性在内部自适应呈现

3 height

取值 说明
auto（默认） 浏览器将会为指定的元素计算并选择一个高度.
<length> 指定值
<percentage> 指定为包含块高度的百分比
fill-available 充分利用可用空间
fit-content 收缩到包裹内容
min-content 收缩到最小
max-content 最大内容高度

4 min-width/max-width、min-height/max-height

1 与众不同的初始值

  • width / height 的默认值是 auto
  • min-width / min-height 的默认值是auto（w3c/mdn上是0）
  • max-width / max-height的默认值是none

2 权重（width / height 规则相同）

  1. width，min-width，max-width只有其中一个，则最终宽度为其值

  2. 有width 和 max-width，则最终宽度不能大于max-width

    • width < max-width 时，最终宽度为width的值
    • width > max-widht 时，最终宽度为max-width的值

  3. 有width 和 min-width，则最终宽度不能小于min-width

    • width > min-width 时，最终宽度为width的值

    • width < min-widht 时，最终宽度为min-width的值

  4. 有min-width和max-width，则最终宽度为二者中较大的那个值

    • min-width < max-width 时，则最终宽度为max-width的值

    • min-width > max-width 时，则最终宽度为min-width的值

  5. 有width，min-width，max-width

    • min-width > max-width，无论width值如何，最终宽度都是min-width的值

    • min-width < max-width，根据width取值，最终宽度在最小和最大宽度之间

      • width < min-width < max-width，则最终宽度为min-width的值
      • min-width < width < max-width，则最终宽度为width的值
      • min-width < max-width < width，则最终宽度为max-width的值

1.7 总结

  1. 块级元素: margin 上下可能有折叠
  2. 行内非替换元素
    1. width、height、margin-top、margin-bottom 不起作用
    2. padding-top、padding-bottom、border-top、border-bottom 效果比较特殊，会撑大区域但不占据空间

2.2 元素分类

1 按照显示（是否在同一行）分类

1 块级元素

  1. 独占父元素一行，自上而下排列

  2. 宽度、高度和内外边距可以直接设置

    1. 宽度：默认为父元素宽度的 100%
    2. 高度：默认为 0，由内容撑开

  3. 块级元素是一个容器或盒子，可以容纳行内元素或块级元素（p h dt 等文字类块级元素内部不能放块级元素）

2 行内元素

  1. 包裹性：默认宽高就是它本身内容的宽高
  2. 自适应性：在父元素一行显示，从左至右排列，如果一行不足以容纳，会换行
  3. 直接设置宽高无效（不可以设置宽高）
  4. 行内元素一般只包含行内元素
    • <a> 中不能容纳 <a>
    • <a> 中可以容纳块级元素

3 行内块元素

  1. 包裹性：默认宽高就是它本身内容的宽高

  2. 自适应性：在一行显示，从左至右排列（和其它行内或行内块元素之间会有空白缝隙），如果一行不足以容纳，会换行

  3. 宽度、高度和内外边距可以直接设置

  4. 默认行内块元素

2 按照内容（是否会被浏览器替换）分类

1 概述

  1. 根据 “外在盒子” 把元素分为内联元素和块级元素

  2. 根据是否具有可替换内容，把元素分为替换元素和非替换元素

    1. 替换元素

      1. 元素本身没有实际内容，浏览器根据元素的类型和属性，来决定元素具体显示内容，如 img、input、textarea、iframe、video、audio、embed、canvas、object…
      2. 匿名可替换元素：通过 CSS content 属性来插入的对象被称作匿名可替换元素
      3. 所有的替换元素都是内联元素

    2. 非替换元素

      1. 元素本身有实际内容，浏览器直接将其内容显示出来
      2. 块级元素都是不可替换元素，即可以认为替换/非替换是针对内联元素的

2 替换元素的特性

  1. 内容的外观不受页面上的 CSS 的影响

    • 替换元素是一种外部对象，它们外观的渲染，是独立于 CSS 的
    • 样式表现在css作用域之外，如果要替换元素本身的外观，需要类似appearance的属性。或者是浏览器本身暴露的接口。但是直接input[type='checkbox']{}没办法改变内间距背景色等特性

  2. 有自己的尺寸

    • 在 Web 中，很多替换元素在没有明确尺寸设定的情况下，其默认的尺寸（不包括边框）是 300 像素×150 像素，如 <video> <iframe> <canvas>
    • 少部分替换元素为 0 像素，如<img>
  • 表单元素的替换元素的尺寸则和浏览器有关，没有明显的规律

  1. 在很多 CSS 属性上有自己的一套表现规则

    • 如 vertical-align 默认值的 baseline ，但对于替换元素，默认为元素的下边缘

3 替换元素的尺寸计算规则

1 尺寸类型
  1. 固有尺寸
    • 指的是替换内容原本的尺寸
    • 如图片，视频都有自身的尺寸
  2. HTML 尺寸
    • HTML 尺寸只能通过HTML 原生属性改变
    • HTML 原生属性，如<img>的 width 和 height 属性、<input>的 size 属性、<textarea>的 cols 和 rows 属性等
  3. CSS 尺寸
    • 可以通过 CSS 的 width 和 height 或者 max-width/min-width 和max-height/min-height 设置的尺寸
    • 对应盒尺寸中的 content box
2 计算规则
  1. 没有 CSS 尺寸和 HTML 尺寸，则使用固有尺寸作为最终的宽高
  2. 没有 CSS 尺寸，则使用 HTML尺寸作为最终的宽高
  3. 有 CSS 尺寸，则最终尺寸由 CSS 属性决定
  4. 如果固有尺寸含有固有的宽高比，css尺寸和html尺寸如果只设置了宽高中的一个，则另一个值根据固有尺寸宽高比变化
  5. 如果以上都不符合，则最终宽度表现为 300 像素，高度为 150 像素
    • <img> 较为特殊
      • chrome下默认为0*0
      • firefox下默认为0*22，且firfox下如果省略src属性，则<img>不是替换元素，而是一个普通的内联元素
  6. 内联替换元素和块级替换元素使用上面同一套尺寸计算规则
    • 即内联替换元素块级化，计算规则不变
  7. 无法改变替换元素内容的固有尺寸
    • 我们为<img>设置宽高，好像是覆盖了固有尺寸，其实改变的是content box的宽高，而<img>的content box替换内容默认适配方式是填充（fill）
    • css3替换元素的替换内容的适配方式可以通过object-fit 属性修改

三 内联盒模型

3.1 组成

1 内容区域 content area

  1. 文字内容区域可以理解为选中文字时的区域
    1. content area 高度由 font-size 和 font-family 决定
    2. font-family 为 simsun(宋体) 时，content area = font-size
  2. 图片这样的替换元素，内容区域可以看成元素自身

2 内联盒子 inline box

  1. 内联盒子：内联元素都有一个内联盒子
  2. 匿名内联盒子：纯文本
    • 注意：并不是所有光秃秃的文字都是“匿名内联盒子”，关键要看前后的标签是内联还是块级

3 行框盒子 line box

  1. 由连续的“内联盒子”组成的一行就是行框盒子
    1. inline 内联元素形成行框盒子的是内部的文字，如果内部为空，就形不成行框盒子
    2. inline-block 内联元素形成行框盒子的是元素本身，内部为空也能形成行框盒子
  2. 行框盒子的高度是由高度最高的那个“内联盒子”决定

4 空白幽灵节点 strut

  1. 每个 line-box 起始位置都有一个具有该元素的字体和行高属性的 0 宽度的内联盒，此内联盒叫做 strut（支柱）。
  2. 永远透明，无法获取，表现如同文本节点一样，也是个盒子

3.2 行高

1 概念

    1. 文本内联元素的基线就是小写 x 的下边缘

      2. 如果该元素内没有内联元素或者 overflow 不是 visible，其基线就是 margin 的底边缘

  • 此时文字在包含元素中垂直居中

  • 此时文字在包含元素中偏下的位置

middle 使元素的中部与父元素的基线加上父元素x-height的一半对齐
<length> 使元素的基线对齐到父元素的基线之上的给定长度。可以是负数
<percentage> 使元素的基线对齐到父元素的基线之上的给定百分比，该百分比是line-height属性的百分比。可以是负数

2 相对整行垂直对齐

值 说明
top 使元素及其后代元素的顶部与整行的顶部对齐
bottom 使元素及其后代元素的底部与整行的底部对齐

3.3 对几个现象的理解

1

  • inline 内联元素形成行框盒子的是内部的文字，内部为空，没有形成行框盒子，所以 div 的内部 span 高度为 0，则 div 的高度也为 0
  1. inline-block 内联元素形成行框盒子的是元素本身，内部为空也能形成行框盒子，既有了幽灵节点 strut
  2. 虽然 span 的高度还是 0，但幽灵节点 strut 默认是有行高的，所以 div 的高度不为 0

2

  • 底部有空隙的原因
    1. img 形成了行框盒子，所以出现了 strut
    2. vertical-align 默认为 base-line 对齐，strut 的 base-line 为 x 的下边缘，img 的 base-line 为底部

1 在图片前边添加文本现象是一样的

2 解决方式

  1. img 块级化，vertical-align 对块级元素无效（且没有了行框盒子，也就没有了 strut）
  2. 修改 vertical-align 为其他值比如 bottom top middle ...
    • 如 bottom，指的是文字内容区域的下边缘和图片的下边缘，自然就没了空隙
  3. 减小 line-height 的值，也就是减小行间距，会造成文字向下移动

3 单行文本垂直居中

  1. 如果包含块没有设置高度，则只要设置 line-height 即可
  2. 如果包含块设置了高度，则设置 line-height = height

4 多行文本垂直居中

  1. 与单行文本居中，原理一样，使用 span 包含多行文本，将此 span 看成单行文本

  2. 设置包含块 line-height = height

  3. span 设置

    1. display: inline-block
    2. vertical-align: middle
      • 为了让 span 在 div 中居中
    3. 一个合适的 line-height
      • 作用于 span 内部文本的显示

  4. 为何 span 要设置 display: inline-block ?

  • span的display为默认的inline时，形成行框盒子的是span的内容，由于div的line-height会作用于行框盒子的strut，且div的line-height大于span的line-height，所以行框应用的是div的line-height，也就是说span的内容应用的是div的line-height，就会导致文字行距很大
  • span的display设置为line-block时，形成行框盒子的是span本身，span的内部还会形成一个行框盒子，这时span设置的line-height就会对内部的行框盒子起作用，即span应用的是div的line-height，sanp的内容应用的是span自身的line-height，所以文字就能正常显示
